## プラグインの導入

### 導入までの流れ

1. プラグインのダウンロードとインストール
2. プラグインの初期設定
3. cron設定
4. 配信設定
5. コンテンツ設定
6. 動作確認



### 1.プラグインのダウンロードとインストール

* [EC-CUBE OWNERS STORE](http://www.ec-cube.net/owners/)より、プラグインを購入してください。
* 購入が完了すると、EC-CUBE管理画面よりプラグインをインストールできるようになります。手順に沿ってインストールしてください。
* プラグインのインストールが完了するとEC-CUBE管理画面に「オーナーズストア」→「プラグイン一覧」→「かご落ち通知メールプラグイン」が表示されます。

### 2.プラグインの初期設定

* EC-CUBE管理画面より「オーナーズストア」→「プラグイン一覧」→「かご落ち通知メールプラグイン」の設定アイコンをクリックします。


### 3.cron設定

* メールの配信プログラムを定期的に実行するために、サーバにcronの設定をおこなってください。

	実行日時：かご落ち通知メールプラグインの実行プログラムは1日に1回起動するように設定しないと、正しく動作しません。
	          設定する実行時間＝プログラムが起動され、メール送信が開始される時間になります。
	          時間は任意で設定できますが、早朝や深夜に設定しないようご注意ください。
	          設定可能な分/時/日/月/曜日のうち、分と時を設定してください。
	          分＝0/時=17/日=*/月=*/曜日＝*とすると、毎日17時にプログラムが実行されます。
	          
	          
	実行ファイルフルパス：実行ファイルはプラグインをインストールした際に配置されています。
	                      cron設定-cronに設定する実行ファイルのパス表示されている実行ファイルパスを確認してください。
	                      

    * 専用サーバ、VPNサーバ、クラウドサーバ等でroot権限を持っており、コマンドで設定する場合(Linux)
    Linuxコマンドの操作に慣れている方が行ってください。
    設定方法が不明の場合は「cron　設定方法」などで検索して設定を行ってください。
    参照例：
    https://www.express.nec.co.jp/linux/distributions/knowledge/system/crond.html
    https://ja.wikipedia.org/wiki/Crontab
    


    * レンタルサーバなどコントロールパネルから設定をおこなう場合
    各ホスティング会社のヘルプページなどでcronの設定方法が紹介されています。
    説明に従って設定を行ってください。
    
            * 主なホスティングサーバーのヘルプページ

            * XSERVER
            https://www.xserver.ne.jp/manual/man_program_cron.php
            * ヘテムル
            https://support.heteml.jp/hc/ja/articles/360042620493
            * ロリポップ！レンタルサーバー
            https://lolipop.jp/manual/user/cron/
            * さくらインターネット
            https://help.sakura.ad.jp/206053892/
            * お名前.com レンタルサーバー　（コマンドスケジューラ）
            https://www.onamae-server.com/support/guide/gs/controlpanel/utility/command_scheduler.php
            * お名前.com VPSサーバー
            http://guide.onamae-server.com/vps/6/29/69/99/

    ！！！注意！！！
    かご落ち通知メールプラグインのcronは１つだけ設定してください。２つ以上設定すると重複してメールが送信されてしまいます。
    
   
### 4.配信設定

    * 最後にカートが更新されてからメールを送るまでの日数を設定してください。
    ※カートが更新されるケースは、顧客が商品をカートにいれた際と、カートに商品が残っている状態でログインした際です。


### 5.コンテンツ設定

    * メールに表示するカート内商品の最大表示数 (1～10),、おすすめ商品の最大表示数 (0～10)を設定してください。
    ※おすすめ商品は株式会社イーシーキューブ製のおすすめ商品管理プラグイン(https://www.ec-cube.net/products/detail.php?product_id=1757)がインストールされており、かつおすすめ商品が設定されている場合のみ表示されます。

    * メールの件名を変更する場合、変更してください。

    * メール文面内のショップURLを変更したい場合は変更してください。

    
### 6.動作確認


    * 管理者宛て完了メール
    かご落ち通知メールが完了した際、送信完了と送信件数がメールされます。
    
    * テストメール送信
    
    顧客に送信されるメールを、管理者メール宛に送信してメールの内容を確認することが出来ます。
    ※テストメールが正しく送信されても、cronが正しく設定されていないと顧客に送信されないケースがあります。


### 5. メールが配信されない場合
* EC-CUBE管理画面、「設定」→「システム設定」→「ログ表示」→「admin-yyyy-mm-dd.log」のログを確認してください。
内容を見ても解決できない場合は、エラーの内容と現象を弊社までお問い合わせください。
